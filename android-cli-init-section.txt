Он стандартизирует основные компетенции разработчиков для рабочих процессов, ориентированных на агенты, предоставляя точку входа к официальным инструментам, навыкам и знаниям, необходимым для более эффективной разработки. Он также может оптимизировать CI, сопровождение и любую другую автоматизацию на основе скриптов в условиях все более распределенной разработки Android.\n\nНапример, агент или скрипт могут использовать CLI для автоматизации настройки среды, создания новых проектов на основе шаблонов и управления виртуальными устройствами непосредственно с вашего терминала. Это также предоставляет вашим агентам доступ к навыкам Android и специализированной базе знаний Android , что помогает гарантировать, что ваши проекты соответствуют рекомендуемым Android шаблонам и передовым практикам.\n\nУстановите Android CLI.\n\nДля установки Android CLI выполните следующие действия:\n\nЗагрузите Android CLI .\n\nЧтобы убедиться, что вы используете последнюю версию, обновите Android CLI:\n\nandroid update\n\nСовет: Регулярно android update , чтобы быть в курсе последних функций.\n\nЧтобы проверить, установлен ли Android CLI на вашем компьютере, выполните команду which android или command -v android : если она вернет путь, значит, он установлен.\n\nНастройка для агентов\n\nЧтобы помочь агентам понять и использовать Android CLI, запустите init для установки навыка android-cli :\n\nandroid init\n\nИзвестные проблемы\nКоманда android emulator для Windows в настоящее время отключена.\n\nЕсли у вас возникнут какие-либо проблемы или вы захотите оставить отзыв, пожалуйста, сообщите об ошибке .\n\nГлобальные опции\n\nЭто необязательные флаги, которые можно использовать с другими командами командной строки Android.\n\n-h, --help\n\nИспользование: android <command> -h\n\nОписание: Отображает справочное руководство по инструменту или конкретной команде.\n\nПримеры:\n\nandroid -h\nandroid create -h\n--sdk\n\nИспользование: android --sdk=<path-to-sdk> <command>\n\nОписание: Путь к Android SDK, который вы хотите использовать для последующей команды. Вы можете использовать параметр --sdk , чтобы временно переопределить Android SDK по умолчанию, вместо того чтобы каждый раз изменять глобальные переменные среды при переключении. Чтобы проверить, какой Android SDK используется по умолчанию, выполните команду android info .\n\nПример: android --sdk=<path/to/sdk> sdk list\n\nКоманды\n\nВ этом разделе перечислены все команды Android CLI и описано их назначение. Все эти команды должны начинаться с префикса android , например android create , android run и так далее. Необязательные модификаторы заключаются в квадратные скобки [] , а обязательные аргументы — нет.\n\ncreate\n\nИспользование: android create [--dry-run] [--verbose] [--name=<application-name>] [--output=<dest-path>] [<template-name>]\n\nОписание: Инициализация нового проекта на основе шаблона. Чтобы просмотреть параметры шаблона, выполните команду android create -h .`.\n\nАргументы (обязательные):\n\n-o, --output - Путь к целевому каталогу проекта.\n\nПараметры:\n\n--dry-run - Имитирует весь процесс создания проекта без фактического сохранения каких-либо файлов. Например, вы можете выполнить пробный запуск, чтобы посмотреть, как работают различные шаблоны, прежде чем выбрать один из них.\n--verbose - Включает подробный вывод информации, в том числе о том, какие файлы копируются из шаблона.\n--name=<application-name> - Название каталога проекта. Если не указано, используется выходной каталог.\n<template-name> - Название шаблона, на основе которого будет создан новый проект. Если оно опущено, используется empty-activity-agp-9 .\n\nПример: android create --dry-run --verbose empty-activity-agp-9\n\ncreate list\n\nИспользование: android create list\n\nОписание: Список всех доступных шаблонов для создания нового проекта.\n\ndescribe\n\nИспользование: android describe [--project_dir=<project-directory>]\n\nОписание: Анализирует проект Android для генерации описательных метаданных. Эта команда определяет и выводит пути к JSON-файлам, подробно описывающим структуру проекта, включая цели сборки и соответствующие места расположения выходных артефактов (например, APK-файлов). Эта информация позволяет другим инструментам и командам эффективно находить артефакты сборки.\n\nПараметры:\n\n--project_dir - Каталог проекта для описания. Если опущено, используется текущий каталог.\n\nПример: android describe --project_dir=/path/to/your/project\n\ndocs\n\nПрименение:\n\nandroid docs search <query>\nandroid docs fetch <kb-url>\n\nОписание: Команда ` android docs — это двухэтапный процесс доступа к базе знаний Android непосредственно из командной строки. Сначала выполните поиск документации, связанной с вашим запросом, используя команду search . Результаты поиска будут включать специальные URL-адреса, начинающиеся с kb:// , которые затем можно использовать с командой fetch для вывода команд документации в терминал.\n\nПримеры:\n\nandroid docs search 'How do I improve my app performance?'\nandroid docs fetch kb://android/topic/performance/overview\nemulator create\n\nИспользование: android emulator create [--list-profiles] [--profile=<profile-name>]\n\nОписание: Создание виртуального устройства.\n\nПараметры:\n\n--list-profiles - Отображает список профилей устройств, которые можно использовать для создания устройства.\n--profile=<profile-name> - Создать устройство с указанным профилем. Если этот параметр опущен, будет создан профиль medium_phone .\nemulator list\n\nИспользование: android emulator list\n\nОписание: Отображает список доступных виртуальных устройств.\n\nemulator start\n\nИспользование: android emulator start <device-name>\n\nОписание: Запуск указанного виртуального устройства.\n\nАргументы (обязательные):\n\n<device-name> - Название устройства для запуска (например, medium_phone ). Используйте android emulator list , чтобы увидеть доступные устройства.\n\nПример: android emulator start medium_phone\n\nemulator stop\n\nИспользование: android emulator stop <device-serial-number>\n\nОписание: Остановить указанное виртуальное устройство.\n\nАргументы (обязательные):\n\n<device-serial-number> - Серийный номер устройства, которое нужно остановить.\n\nПример: android emulator stop emulator-5554\n\ninfo\n\nИспользование: android info\n\nОписание: Отображает путь к используемому по умолчанию Android SDK. Чтобы изменить используемый Android SDK, используйте --sdk .\n\ninit\n\nИспользование: android init\n\nОписание: Настройте среду для агентов, установив навык android-cli .\n\nlayout\n\nИспользование: android layout [--pretty] [--output] [--diff]\n\nОписание: Возвращает макет пользовательского интерфейса активного приложения Android (подключенного через физическое устройство или эмулятор) в формате JSON.\n\nПараметры:\n\n-p, --pretty - Форматирует вывод JSON с отступами и переносами строк для удобства чтения человеком.\n-o, --output - Указывает местоположение файла для сохранения дерева компоновки. Если не указано, JSON-данные выводятся непосредственно в стандартный поток вывода.\n-d, --diff - Возвращает список только тех элементов макета, которые изменились с момента создания последнего внутреннего снимка (последнего запуска функции компоновки), вместо полного дерева макета.\n\nПример: ` android layout --output=./hierarchy.json\n\nskills add\n\nНавыки Android — это специальные инструкции, разработанные для того, чтобы помочь агентам лучше понимать и выполнять определенные шаблоны, соответствующие передовым методам и рекомендациям по разработке под Android. Для получения дополнительной информации см. раздел «Введение в навыки Android» .\n\nИспользование: android skills add [--all] [--agent=<agent-name>] [--skill=<skill-name>]\n\nОписание: Устанавливает навыки Android в каталоги навыков для всех обнаруженных агентов. Если у вас нет существующих каталогов агентов и вы не указываете конкретных агентов, навыки для Gemini и Antigravity будут установлены в ~/.gemini/antigravity/skills .\n\nПараметры:\n\n--all - Добавить все навыки Android одновременно. Если этот параметр опущен, навык будет установлен для всех обнаруженных агентов.\n--skill - Название навыка, который вы хотите установить. Если оно опущено (и не указано --all ), будет установлен только навык android-cli .\n\nПример: android skills add --agent='gemini' edge-to-edge\n\nskills find\n\nИспользование: android skills find <string>\n\nОписание: Найти навыки, соответствующие заданной строке.\n\nАргументы (обязательные):\n\nstring — Строка, соответствующая описанию навыка.\n\nПример: команда android skills find 'performance'\n\nskills list\n\nИспользование: android skills list [--long]\n\nОписание: Перечислите доступные навыки.\n\nПараметры:\n\n--long - Вывести дополнительную информацию для каждого навыка, включая описание навыка и для каких агентов он уже установлен.\nskills remove\n\nИспользование: android skills remove [--agent] --skill=<skill-name>\n\nОписание: Удаление навыка. Если вы не укажете конкретных агентов, навык будет удален для всех агентов.\n\nАргументы (обязательные):\n\n--skill - Название навыка, который нужно удалить.\n\nПараметры:\n\n--agent - Список агентов, разделенных запятыми, для которых нужно удалить навык. Если этот параметр опущен, навык будет удален для всех агентов.\n\nПример: android skills remove --agent='gemini' --skill=edge-to-edge\n\nscreen capture\n\nИспользование: android screen capture [--output] [--annotate]\n\nОписание: Делает снимок экрана подключенного устройства.\n\nПараметры:\n\n-o, --output - Указывает местоположение файла для сохранения снимка экрана. Если этот параметр опущен, необработанные данные PNG будут выведены непосредственно в стандартный поток вывода.\n-a, --annotate - Рисует ограничивающие рамки с подписями вокруг всех элементов пользовательского интерфейса, обнаруженных на изображении, для использования с командой resolve .\n\nПример: android screen capture --output=ui.png\n\nscreen resolve\n\nИспользование: android screen resolve --screenshot=<path> --string=<string>\n\nОписание: Преобразует визуальные метки с аннотированного скриншота, полученного с помощью screen capture , в фактические координаты экрана (x, y). Полезно для создания скриптов для обработки кликов по элементам без необходимости вручную вычислять их положение.\n\nФлаги:\n\n--screenshot - Путь к скриншоту с аннотацией.\n--string - Строка, содержащая как минимум один заполнитель, соответствующий метке элемента пользовательского интерфейса в формате #<number> . Часть #<number> будет заменена экранными координатами.\n\nПример:\n\nЕсли метка 5 находится в координатах (500, 1000), то команда\n\nandroid screen resolve --screenshot=ui.png --string=\"input tap #5\"\n\n\nвозвращает результат\n\ninput tap 500 1000\n\nsdk install\n\nИспользование: android sdk install <package[@version]> [--beta] [--canary] [--force]\n\nОписание: Установить указанный(е) пакет(ы) SDK.\n\nАргументы (обязательные):\n\npackage[@version] — Список пакетов для установки, разделённых пробелами. Если версия не указана, устанавливается последняя версия пакета из канала (по умолчанию — стабильный канал).\n\nПараметры:\n\n--beta - Включить бета-версии пакетов.\n--canary - Включить тестовые пакеты.\n--force - Принудительное возвращение к более старой версии.\n\nПримеры:\n\nandroid sdk install platforms/android-34 build-tools/34.0.0 - Установите последние версии пакетов Android SDK Platform 34 и SDK Built Tools 34.0.0 из стабильного канала.\nandroid sdk install platforms/android-34@2 - Установка версии 2 пакета Android SDK Platform 34.\nandroid sdk install --canary system-images/android-35/google_apis/x86_6 - Установите последнюю версию образа системы Android 35 из канала canary.\nandroid sdk install --force platforms/android-33@1 - Возврат к версии 1 пакета Android SDK Platform 33 из стабильного канала.\nsdk list\n\nИспользование: android sdk list <package-pattern>\n\nОписание: Список установленных и доступных пакетов SDK.\n\nАргументы (обязательные):\n\n<package-pattern> - Фильтрация пакетов по шаблону.
